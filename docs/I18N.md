# wzrdVID UI Localization Notes

This localization layer is UI/readability only. It does not translate user media, rendered video, subtitles, filenames, presets as creative algorithms, or any content intelligence.

## Desktop App

- Desktop UI strings live in `app_i18n.py`.
- The PySide6 app stores the selected UI language as `ui_language` in the existing local settings JSON.
- `system` means the app resolves the OS/browser language when it can.
- Missing keys fall back to English, then to the stable string key.
- Non-English strings are draft translations and should be reviewed by fluent speakers.
- The current desktop layer localizes the main shell, groups, primary controls, status labels, update/reset copy, file-dialog titles, common tooltips, table type labels, and the most common user-facing warning/dialog copy.
- Some preset/effect option names, preset descriptions, raw ffmpeg/probe/render errors, debug details, and lower-level render/log messages intentionally remain stable English internals for now.

## Public Site and Lite

- Static site and Lite strings live in `docs/i18n.js`.
- `docs/index.html` and `docs/lite/index.html` use `data-i18n` attributes for visible text.
- Lite runtime messages use `window.WZRD_I18N.t(...)` from `docs/i18n.js`.
- The selected site/Lite language is stored only in `localStorage` as `wzrdvid.uiLanguage`. There are no accounts, cookies, analytics, uploads, network calls, or remote config.
- Arabic sets `dir="rtl"` at runtime. This is structural RTL support, not a claim of perfect RTL visual QA.
- The homepage, Lite, and runtime Lite status/log strings share the same static fallback layer. English is complete for the current UI surface; non-English strings remain draft and partial where a stable English fallback is safer than guessing.

## Adding A Language

1. Add the language code and native label to `SUPPORTED_LANGUAGES` in `app_i18n.py`.
2. Add the same language code and label to `LANGUAGES` in `docs/i18n.js`.
3. Add draft strings under `APP_STRINGS` / `DRAFTS`.
4. Keep stable keys. Do not use English source text as keys.
5. Run the UI/content validation matrix from `AGENTS.md`.
